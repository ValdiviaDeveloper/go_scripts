# Guía de Contribución

¡Gracias por interesarte en contribuir! Aquí está como puedes ayudar.

## 📋 Antes de empezar

- Lee [CODE_OF_CONDUCT.md](./CODE_OF_CONDUCT.md)
- Asegúrate que el código compila: `cd portscanner && go build`
- Las contribuciones deben cumplir con leyes y regulaciones

## 🔄 Proceso de contribución

### 1. Fork y crea una rama
```bash
git clone https://github.com/TU_USERNAME/go_scripts.git
cd go_scripts
git checkout -b feature/tu-feature
```

### 2. Haz tus cambios
- Código limpio y bien documentado
- Sigue el estilo del proyecto
- Prueba localmente

### 3. Commit descriptivos
```bash
git commit -m "feat: agregar nueva funcionalidad"
git commit -m "fix: corregir bug en escaneo"
git commit -m "docs: mejorar documentación"
```

### 4. Push y abre Pull Request
```bash
git push origin feature/tu-feature
```

## ✅ Checklist para PRs

- [ ] El código compila sin errores
- [ ] Los cambios son claros y documentados
- [ ] Actualizaste README si es necesario
- [ ] Testeaste en múltiples plataformas si es posible
- [ ] Cumples con el CODE_OF_CONDUCT

## 🐛 Reportar bugs

Usa [GitHub Issues](../../issues) con:
- Descripción clara
- Pasos para reproducir
- Sistema operativo y versión
- Versión del software

## 💡 Sugerir mejoras

- Crea un issue con etiqueta `enhancement`
- Describe el problema y tu idea
- Explica el valor agregado

## 📖 Mejorando documentación

- Abre un PR con cambios al README o guías
- Sé claro y conciso
- Verifica sintaxis Markdown

## 🚀 Tipos de contribución bienvenidos

- 🐛 Corrección de bugs
- ✨ Nuevas características
- 📚 Mejoras a documentación
- 🎨 Mejoras de UI/UX
- ♻️ Refactoring de código
- ⚡ Optimizaciones de rendimiento
